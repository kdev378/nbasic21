# 付録 B エラーメッセージ辞典 — 困ったときはここ

エラーには 2 種類あります。

- **コンパイルエラー** — 翻訳の段階で見つかる文法の間違い。
  `ファイル名:行:桁: error: 説明` の形で出ます。**行番号の場所を
  見れば必ず原因がある** (か、その 1〜2 行上にある) のが特徴。
- **実行時エラー** — 動かしている最中に起きる問題。
  `?RUNTIME ERROR: 説明` の形で出て、プログラムが止まります。

## B.1 コンパイルエラー

| メッセージ | 原因と直し方 |
|---|---|
| 不正な文字です | **全角文字**の混入が 9 割。全角スペース・`＋`・`（` などを半角に。日本語が使えるのは `"..."` の中とコメントだけ |
| 文字列リテラルが閉じていません | `"` の閉じ忘れ。行末までに `"` を |
| 式が必要ですが 行末 があります | `PRINT 1 +` のように式が途切れた。演算子の右側を確認 |
| 文の終わりが必要ですが '...' があります | 1 つの文の後ろに余計なものが続いている。命令のつづり間違いも疑う |
| 'NEXT' が必要です (FOR に対応) | FOR と NEXT の数が合っていない。ブロックの対応を確認 |
| 'END IF' が必要です | ブロック IF を閉じていない。1 行 IF のつもりなら THEN の後ろを同じ行に |
| NEXT の変数 '...' が FOR の変数と一致しません | 二重ループの NEXT を逆順に書いていないか (内側を先に閉じる) |
| 変数 '...' は宣言済みです | 同じ名前を 2 回 DIM した |
| ラベル '...' が重複しています | 同じ行番号・ラベル名を 2 回使った |
| ラベル '...' が〜ありません | GOTO/GOSUB の飛び先がない。つづりを確認。SUB の中から外のラベルへは飛べない |
| 代入: INTEGER を STRING に変換できません | 型違いの代入。`A$ = 5` はダメ、`A$ = STR$(5)` または `A$ = "5"` |
| '...' は数値にのみ使えます | 文字列に算術演算子を使った。数にするなら `VAL()` |
| 配列 '...' は N 次元です | 添字の個数が DIM と合っていない |
| '...' という配列・関数はありません | DIM の前に配列を使ったか、関数名のつづり間違い |
| SUB '...' は式の中では呼び出せません | 値を返すのは FUNCTION。SUB は文として呼ぶ |
| EXIT FOR が対応するループの外にあります | EXIT の種類とループの種類が合っているか |

## B.2 実行時エラー

| メッセージ | 原因と直し方 |
|---|---|
| Division by zero | 0 で割った (`\` か `MOD`)。割る前に 0 かどうか IF で確認 |
| Subscript out of range | 配列の範囲外。`DIM A%(9)` なら使えるのは 0〜9。ループの端 (TO の値) を見直す |
| Array used before DIM | DIM する前に配列を使った。DIM は使う場所より先に実行される必要がある |
| Illegal function call | 関数にありえない値を渡した: `SQR(負数)`、`CHR$(300)`、`MID$` の位置 0 など |
| RETURN without GOSUB | GOSUB していないのに RETURN に到達。END を書き忘れてサブルーチンに「落ちて」いないか |
| Out of DATA | READ の回数が DATA の個数を超えた |
| Out of input (EOF) | 入力が尽きた。`WHILE NOT EOF(0)` で守る (第 13 章) |
| File not found | FOR INPUT で開くファイルが無い。ファイル名と場所 (実行したフォルダ) を確認 |
| File already open | 同じ番号を二重に OPEN。CLOSE を忘れていないか |
| File not open | OPEN していない番号に PRINT # / INPUT # した |
| Input past end of file | ファイルの終端を越えて読んだ。`WHILE NOT EOF(n)` で守る |

## B.3 エラーは出ないけど動きがおかしいとき

いちばん難しいのはこれです。**バグ取りの基本 3 手**:

1. **途中経過を PRINT する** — 疑わしい場所に
   `PRINT "ここまで来た A%="; A%` を差し込み、頭の中の想像と
   実際の値のズレを見つける。ズレた場所の直前に犯人がいます。
   (終わったら消すのを忘れずに)
2. **小さくして試す** — データを 100 件から 3 件に減らす、
   ループを 1 回だけにする。小さくても再現するなら、それが
   観察しやすい標本です。
3. **人に説明する** — 声に出して 1 行ずつ説明してみると、
   説明に詰まった行が怪しい。聞き手はぬいぐるみでも OK
   (プロも「ラバーダック・デバッグ」と呼んで実践しています)。

よくある「エラーの出ないバグ」:

| 症状 | 定番の犯人 |
|---|---|
| 合計や個数が 1 回分ずれる | ループの範囲 (`TO N%` と `TO N%-1` の取り違え)。境界を紙で数える |
| 合計が 2 回目の実行で倍になる (ように見える) | 累積変数を 0 に戻し忘れてループの外に置いた |
| IF がぜんぶ同じ側に落ちる | `=<` と書いた? 条件の向き? 数値と文字列を比べていない? |
| 無限ループ | WHILE の条件が永遠に真。ループ内で条件の変数を変えているか確認。止めるのは Ctrl+C |
| 乱数が毎回同じ | RANDOMIZE を書いていない (わざとの場合もある: 第 11 章) |
| 保存したはずのファイルが空 | CLOSE を忘れた |
