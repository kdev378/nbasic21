# 第 16 章 プロジェクト②: ToDo 管理アプリ

最終章です。今度は「ゲーム」ではなく「データを預かる実用アプリ」を
作ります。題材は ToDo (やること) リスト。地味に見えて、
**データ構造 + 画面 + ファイル保存**という、実用ソフトの 3 要素が
全部入っています。そして今回は、プロがやるように
**仕様を決めるところから**始めます。

## 16.1 仕様を決める — 作る前に「何を作るか」

紙に書き出します。これだけで完成率が段違いに上がります:

> **ToDo アプリ 仕様書 (第 1 版)**
> - やることを何件も登録できる (上限 100 件)
> - 各項目は「内容」と「済んだかどうか」を持つ
> - できること: 一覧を見る / 追加する / 済みにする / 削除する / 終了
> - 終了時に自動でファイルに保存し、起動時に自動で読み込む
> - 操作はメニュー方式 (番号を選ぶ)。機械オンチの家族でも使えること

最後の 1 行のような「誰のためか」も立派な仕様です。

## 16.2 設計 — データと部品を決める

**データ**: 項目は「内容 (文字列)」と「済フラグ (整数)」の組。
第 10 章の練習 4 でやった**ペアの配列**方式でいきます:

```text
TASK$(i)  … i 番の内容            例: "牛乳を買う"
DONE%(i)  … i 番が済みなら -1     (真理値の -1/0 をそのまま使う)
N%        … いま何件あるか
```

**ファイル形式**: 1 行 1 項目の CSV で `済フラグ,内容`:

```text
0,牛乳を買う
-1,本を返す
```

**部品**: やることが 5 つあるので、メイン (メニューのループ) +
処理ごとの SUB という構成に。ここまで決まれば、あとは各部品を
順に埋めるだけです。

## 16.3 実装

```basic
' todo.bas — ToDo 管理アプリ
' 起動: tools/nbc run work/todo.bas
CONST MAXN% = 100
CONST FILE$ = "todo.csv"

' ---- 共有データ (先頭で DIM したものは SUB からも見える: 第 8 章) ----
DIM TASK$(MAXN%)
DIM DONE%(MAXN%)
DIM N% AS INTEGER

' ==== 起動: 保存データの読み込み ====
LoadTasks

' ==== メインループ: メニューを出して選ばせる ====
DO
  ShowList
  PRINT
  COLOR 14
  PRINT "[1]追加  [2]済みにする  [3]削除  [0]保存して終了"
  COLOR 7
  INPUT "番号を選んでください"; C%
  SELECT CASE C%
    CASE 1
      AddTask
    CASE 2
      MarkDone
    CASE 3
      DeleteTask
    CASE 0
      SaveTasks
      PRINT "保存しました。おつかれさま!"
      EXIT DO
    CASE ELSE
      PRINT "0〜3 で選んでください"
      SLEEP 1
  END SELECT
LOOP
END

' ======================================================
' 一覧表示 (画面を消して描き直す)
' ======================================================
SUB ShowList
  CLS
  COLOR 11
  PRINT "====== やることリスト ("; N%; "件) ======"
  COLOR 7
  IF N% = 0 THEN
    PRINT "  (何もありません。えらい!)"
  END IF
  FOR I% = 0 TO N% - 1
    IF DONE%(I%) THEN
      COLOR 8                       ' 済んだものは灰色で
      PRINT I% + 1; ": [済] "; TASK$(I%)
      COLOR 7
    ELSE
      PRINT I% + 1; ": [  ] "; TASK$(I%)
    END IF
  NEXT I%
END SUB

' ======================================================
' 追加
' ======================================================
SUB AddTask
  IF N% >= MAXN% THEN
    PRINT "満杯です! 何か片付けてから来てください"
    SLEEP 2
    EXIT SUB
  END IF
  INPUT "やることを入力"; T$
  IF T$ = "" THEN EXIT SUB          ' 空入力はキャンセル扱い
  TASK$(N%) = T$
  DONE%(N%) = 0
  N% = N% + 1
END SUB

' ======================================================
' 済みにする / 削除 — 番号を聞く部分は共通なので関数に
' ======================================================
FUNCTION AskNumber% ()
  INPUT "何番ですか"; K%
  IF K% < 1 OR K% > N% THEN
    PRINT "その番号はありません"
    SLEEP 1
    AskNumber% = 0                  ' 0 = 無効、を返す約束にする
  ELSE
    AskNumber% = K%
  END IF
END FUNCTION

SUB MarkDone
  K% = AskNumber%()
  IF K% = 0 THEN EXIT SUB
  DONE%(K% - 1) = -1                ' 表示は 1 始まり、配列は 0 始まり
END SUB

SUB DeleteTask
  K% = AskNumber%()
  IF K% = 0 THEN EXIT SUB
  ' 消した穴を詰める: K% 番より後ろを 1 つずつ前へ (第 6 章の「ずらし」)
  FOR I% = K% - 1 TO N% - 2
    TASK$(I%) = TASK$(I% + 1)
    DONE%(I%) = DONE%(I% + 1)
  NEXT I%
  N% = N% - 1
END SUB

' ======================================================
' 保存と読み込み (第 12 章の CSV そのまま)
' ======================================================
SUB SaveTasks
  OPEN FILE$ FOR OUTPUT AS #1
  FOR I% = 0 TO N% - 1
    PRINT #1, DONE%(I%); ","; TASK$(I%)
  NEXT I%
  CLOSE #1
END SUB

SUB LoadTasks
  N% = 0
  ' 初回起動 (ファイルがまだ無い) に備え、まず OUTPUT で
  ' 触って APPEND で閉じる…のは面倒なので、素直な手を使う:
  ' 「無ければ空で作る」
  OPEN FILE$ FOR APPEND AS #1
  CLOSE #1
  OPEN FILE$ FOR INPUT AS #1
  WHILE NOT EOF(1) AND N% < MAXN%
    INPUT #1, D%, T$
    IF T$ <> "" THEN
      DONE%(N%) = D%
      TASK$(N%) = T$
      N% = N% + 1
    END IF
  WEND
  CLOSE #1
END SUB
```

## 16.4 読みどころ解説

- **共有データは最小限に** — TASK$/DONE%/N% の 3 つだけを共有し、
  あとはすべて部品の中のローカル変数。第 8 章で話した
  「共有はどうしてもの分だけ」の実践です。
- **AskNumber% の「0 = 無効」方式** — 部品が「失敗」を呼び出し元に
  伝えたいとき、ありえない値を返り値にする手法は定番です
  (第 9 章の FOUND% = -1 と同じ発想)。
- **表示は 1 始まり、配列は 0 始まり** — 人間には 1 番から見せ、
  内部では 0 番から数える。境目の `K% - 1` は初心者が最も
  バグらせる場所なので、変換は一か所に集めましょう。
- **削除は「ずらして詰める」** — 第 6 章の配列ずらしが、
  ゲーム (ヘビの体) に続いてここでも主役です。
- **「無ければ空で作る」** — APPEND で開いてすぐ閉じると、
  ファイルが無いときだけ空ファイルができます。初回起動問題の
  小さな定石です。

## 16.5 改造アイデア

1. 「済み」を一括削除する [4] メニューを足す
2. 項目に期限 (`0402` のような日付) を足し、一覧を期限順に
   ソートして表示する (第 10 章のペア SWAP!)
3. 起動時に「未完了が N 件あります」と挨拶させる
4. 第 13 章と合体: `todo add 牛乳を買う` のように引数だけで
   追加できるモードを作る (`COMMAND$(1) = "add"` を見て分岐)
