# 第 6 章 たくさん覚える — 配列

テストの点数を 5 人分覚えたいとき、`A`、`B`、`C`、`D`、`E` と
5 個の変数を作りますか? では 100 人分なら? 1 万人分なら?
そのための道具が**配列**です。配列とループが手を組んだ瞬間、
プログラムは「データ処理」ができるようになります。

## 6.1 配列は「番号つきロッカー」

配列は、**同じ名前で番号だけが違う箱の列**です。
学校のロッカーを思い浮かべてください。

```basic
DIM SCORE%(4)          ' 0 番から 4 番まで、5 個のロッカーを用意

SCORE%(0) = 80
SCORE%(1) = 65
SCORE%(2) = 92
SCORE%(3) = 71
SCORE%(4) = 58

PRINT SCORE%(2)        ' 2 番ロッカーの中身
```

```text
 92
```

- `DIM 名前(上限)` で配列を用意します。**番号は 0 から始まる**ので、
  `DIM SCORE%(4)` で 0〜4 の 5 個できます。
- 番号 (**添字**といいます) を変えて、それぞれの箱を読み書きします。
- 用意した範囲の外 (この例で `SCORE%(5)`) を触ると、実行時に
  「Subscript out of range (添字が範囲外)」というエラーで止まります。

## 6.2 添字に「変数」が使えるのが革命

配列の真の力はここからです。**添字には計算式や変数が使えます**:

```basic
DIM SCORE%(4)
FOR I% = 0 TO 4
  INPUT "点数を入れて"; SCORE%(I%)
NEXT I%

' 全員の合計と平均
TOTAL% = 0
FOR I% = 0 TO 4
  TOTAL% = TOTAL% + SCORE%(I%)
NEXT I%
PRINT "合計:"; TOTAL%; " 平均:"; TOTAL% / 5
```

ループカウンタ I% がそのまま「今何番のロッカーを見ているか」に
なっています。5 人を 1 万人に変えたくなったら、直すのは数字
2 か所だけ。変数 1 万個作戦では不可能だったことです。

## 6.3 定番ワザ: 最大値・最小値を探す

「一番高い点数はどれ?」— 人間は一目で分かりますが、
コンピュータは 1 個ずつしか見られません。そこでこう考えます:

> 「暫定チャンピオン」の箱を用意して、全員と順に対戦させる。
> 勝った (大きかった) ら王座を交代する。

```basic
DIM SCORE%(4)
SCORE%(0) = 80 : SCORE%(1) = 65 : SCORE%(2) = 92
SCORE%(3) = 71 : SCORE%(4) = 58

MAX% = SCORE%(0)              ' 最初の 1 人を暫定チャンピオンに
FOR I% = 1 TO 4
  IF SCORE%(I%) > MAX% THEN
    MAX% = SCORE%(I%)         ' 王座交代!
  END IF
NEXT I%
PRINT "最高点:"; MAX%
```

```text
最高点: 92 
```

(`:` は「同じ行に複数の命令を書く」区切りです。データの用意など
短い行を詰めるときに便利。)

この「暫定チャンピオン方式」は Python でも C でも同じ発想で書く、
正真正銘の基本アルゴリズムです。最小値なら `>` を `<` に変えるだけ。

## 6.4 定番ワザ: 数える

「60 点未満は何人?」のような**条件つきカウント**:

```basic
DIM SCORE%(4)
SCORE%(0) = 80 : SCORE%(1) = 65 : SCORE%(2) = 92
SCORE%(3) = 71 : SCORE%(4) = 58

COUNT% = 0
FOR I% = 0 TO 4
  IF SCORE%(I%) < 60 THEN COUNT% = COUNT% + 1
NEXT I%
PRINT "60 点未満:"; COUNT%; "人"
```

累積パターン (第 5 章) の変形です。「足し込む」のが値ではなく
「1 (= 数える)」になっただけ。

## 6.5 集計表を作る — 添字の逆転の発想

サイコロを 1000 回振って、each 目が何回出たか数えます。
ここで頭をひねる発想が出てきます: **出た目そのものを添字に使う**。

```basic
DIM COUNT%(6)                 ' COUNT%(1)〜(6) を出た目の回数に使う
RANDOMIZE
FOR I% = 1 TO 1000
  DICE% = INT(RND * 6) + 1    ' 1〜6 の乱数
  COUNT%(DICE%) = COUNT%(DICE%) + 1
NEXT I%

FOR ME% = 1 TO 6
  PRINT ME%; "の目:"; COUNT%(ME%); "回"
NEXT ME%
```

3 が出たら `COUNT%(3)` を 1 増やす。IF を 6 個並べる必要は
ありません。この「値を添字にする」テクニックは、成績分布、
文字の出現頻度、投票集計 — あらゆる集計処理の芯になります。

## 6.6 2 次元配列 — 表を覚える

配列は「表」の形にもできます。添字が 2 つになります:

```basic
' 3 クラス × 4 教科の平均点表
DIM AVG(2, 3)                 ' AVG(クラス 0〜2, 教科 0〜3)
FOR C% = 0 TO 2
  FOR S% = 0 TO 3
    AVG(C%, S%) = 50 + C% * 10 + S%   ' 仮のデータを詰める
  NEXT S%
NEXT C%
PRINT "2 組の 3 番目の教科:"; AVG(1, 2)
```

2 次元配列を全部なめるときは二重ループ (第 5 章) がぴったり
はまります。ゲームの盤面 (オセロ、迷路) も 2 次元配列で表します —
第 15 章のスネークゲームで実際に使います。

## 6.7 DATA 文 — プログラムにデータを埋め込む

テストのたびに 5 人分の点数を手で打つのは面倒です。
`DATA` 文を使うと、データをプログラムに埋め込んでおけます:

```basic
DIM SCORE%(4)
FOR I% = 0 TO 4
  READ SCORE%(I%)             ' DATA から順に 1 個ずつ取り出す
NEXT I%
PRINT "3 番目:"; SCORE%(2)

DATA 80, 65, 92, 71, 58
```

`READ` を実行するたびに、`DATA` の並びから次の値がひとつ取り出され
ます。データがプログラムと一緒に保存されるので、実験や練習に
便利です (本格的なデータ保存は第 12 章のファイルで)。

## 他の言語では

```text
NBASIC-21 :  DIM A%(9)     A%(3) = 5
Python    :  a = [0]*10    a[3] = 5
JavaScript:  let a = new Array(10).fill(0); a[3] = 5
C         :  int a[10];    a[3] = 5;
```

「0 番から始まる」「範囲外はエラー (または大事故)」も含めて、
配列の感覚はそのまま他言語に持っていけます。
