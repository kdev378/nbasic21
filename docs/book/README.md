# プログラミングをはじめよう — NBASIC-21 で学ぶ、コンピュータへの命令の書き方

これは、**プログラミングを一度もやったことがない人**のための教科書です。
パソコンの操作に自信がなくても大丈夫。「ファイルを作る」「文字を打つ」
ところから、ゲームや実用ツールを自分の手で作れるようになるところまで、
一歩ずつ進みます。

使う言語は **NBASIC-21**。1970〜80 年代に世界中の初心者がプログラミングを
覚えた言語 BASIC の、現代版です。BASIC は「初心者のための言語」として
生まれました (名前の B は Beginner の B です)。命令が英語の単語そのままで、
書いたことがそのまま起きる。最初の 1 冊に、これほど向いた言語はありません。

そして安心してください — この本で覚える考え方 (変数、くり返し、条件分岐、
関数、アルゴリズム) は、**Python でも JavaScript でも C でも、そっくり
そのまま通用します**。言語は道具にすぎず、考え方こそが本体だからです。
各章の「他の言語では」コラムで、同じことが他の言語でどう書けるのかも
紹介します。

## この本の読み方

- **手を動かしてください。** プログラムは読むだけでは覚えられません。
  スポーツと同じで、自分で書いて、動かして、失敗して覚えます。
  本文のプログラムはすべて実際に動くので、必ず打ち込んで実行して
  ください (コピー & ペーストでも構いませんが、一度は手で打つと
  発見があります)。
- **エラーは友だちです。** 赤い文字が出ても壊れていません。エラーは
  コンピュータからの「ここが分からないよ」という質問です。
  [付録 B](appendix-b-errors.md) にエラーの読み方をまとめてあります。
- 章末の**練習問題**は飛ばさないでください。1 問でいいのでやって
  ください。答えは [巻末](answers.md) にあります。
- 分からないところがあったら、**先に進んでから戻ってくる**のも
  立派な読み方です。

## 表記の約束

- `このような字` はプログラムやコマンド (キーボードで打つもの) です。
- プログラム例の中の `'` から行末まではコメント (説明書き) で、
  打たなくても動きます。
- 「ターミナル」と書いたら、Windows では「コマンドプロンプト」、
  Linux では「端末」のことです。第 1 章で説明します。
