# 第 4 章 分かれ道 — 条件分岐

ここまでのプログラムは一本道でした。この章では「もし〜なら A、
さもなくば B」という**分かれ道**を作ります。これができると、
プログラムが急に「賢く」見えはじめます。

## 4.1 IF — もし〜なら

```basic
INPUT "点数は"; SCORE%
IF SCORE% >= 60 THEN
  PRINT "合格!"
END IF
PRINT "おつかれさまでした"
```

```text
点数は? 75
合格!
おつかれさまでした
```

`IF 条件 THEN` から `END IF` までが分かれ道です。
条件が成り立つときだけ、間に挟まれた行が実行されます。
成り立たないときは、間を**まるごと飛ばして** `END IF` の次に進みます
(だから「おつかれさまでした」はどちらでも表示されます)。

条件に使える比較の記号は 6 つ:

| 記号 | 意味 | | 記号 | 意味 |
|---|---|---|---|---|
| `=` | 等しい | | `<>` | 等しくない |
| `<` | より小さい | | `<=` | 以下 |
| `>` | より大きい | | `>=` | 以上 |

「以上・以下」の `>=` `<=` は、数学の ≧ ≦ を半角 2 文字で書いた
ものです。`=<` のように逆順に書くとエラーになるので注意。

## 4.2 ELSE — さもなくば

「成り立たないときはこっち」も書けます:

```basic
INPUT "点数は"; SCORE%
IF SCORE% >= 60 THEN
  PRINT "合格!"
ELSE
  PRINT "残念、あと"; 60 - SCORE%; "点"
END IF
```

さらに `ELSEIF` で分かれ道を何段でも作れます。
**上から順に調べて、最初に成り立ったところだけ**が実行されます:

```basic
INPUT "点数は"; SCORE%
IF SCORE% >= 80 THEN
  PRINT "評価: A"
ELSEIF SCORE% >= 60 THEN
  PRINT "評価: B"
ELSEIF SCORE% >= 40 THEN
  PRINT "評価: C"
ELSE
  PRINT "評価: 不可"
END IF
```

75 点なら、最初の `>= 80` は不成立、次の `>= 60` が成立するので
「B」。それより下は見もしません。この「上から順に、最初のひとつ」
という動きはとても重要です。もし順番を逆 (40 から) に書いたら
どうなるか、考えてみてください (75 は `>= 40` なので C に
なってしまいます!)。

## 4.3 条件を組み合わせる — AND / OR / NOT

「〜かつ〜」「〜または〜」は `AND` と `OR` で書きます:

```basic
INPUT "年齢は"; AGE%
IF AGE% >= 13 AND AGE% <= 19 THEN
  PRINT "ティーンエイジャーですね"
END IF
IF AGE% < 6 OR AGE% >= 65 THEN
  PRINT "入場無料です"
END IF
IF NOT (AGE% >= 20) THEN
  PRINT "未成年です"
END IF
```

- `A AND B` — A と B の**両方**が成り立つとき
- `A OR B` — A と B の**どちらか** (両方でも) が成り立つとき
- `NOT A` — A が成り立た**ない**とき

`NOT` の例のように、条件をかっこで囲むと読みやすくなります。

> **コラム: 真と偽の正体**
> 実は比較の結果は数値です。成り立つと `-1`、成り立たないと `0`。
> `PRINT 5 > 3` とすると `-1` が表示されます。IF は「0 以外なら
> 進む」という決まりで動いています。多くの言語では真が `1` や
> `true` ですが、考え方は同じです。

## 4.4 短い書き方 — 1 行 IF

やることが 1 つだけなら、1 行にまとめられます:

```basic
INPUT "数を入れて"; N%
IF N% MOD 2 = 0 THEN PRINT "偶数" ELSE PRINT "奇数"
```

`N% MOD 2` は「2 で割った余り」でしたね。余りが 0 なら偶数。
この「MOD で割り切れるか調べる」は定番テクニックです。

## 4.5 SELECT CASE — たくさんの分かれ道をきれいに

同じ変数をたくさんの値と比べるなら、`SELECT CASE` のほうが
すっきり書けます:

```basic
INPUT "今日は何曜日 (1=月 〜 7=日)"; D%
SELECT CASE D%
  CASE 1, 2, 3, 4
    PRINT "平日です。がんばりましょう"
  CASE 5
    PRINT "金曜日! あと少し"
  CASE 6, 7
    PRINT "週末です。ゆっくり休んで"
  CASE ELSE
    PRINT "1〜7 で入れてください"
END SELECT
```

`CASE` には値の並び (`1, 2, 3`)、範囲 (`CASE 1 TO 4`)、
条件 (`CASE IS >= 5`) が書けます。どれにも当てはまらないときは
`CASE ELSE`。IF の連段と同じく、**上から最初に当てはまった 1 つだけ**
が実行されます。

## 4.6 文字列の比較

文字列も `=` や `<` で比べられます:

```basic
INPUT "合言葉をどうぞ"; PW$
IF PW$ = "やまびこ" THEN
  PRINT "ようこそ!"
ELSE
  PRINT "ちがいます"
END IF
```

`<` や `>` は辞書順 (正確には文字コード順) の比較になります。
`"APPLE" < "BANANA"` は成り立ちます。

## 4.7 実践: じゃんけん判定

乱数 `RND` を先取りして、コンピュータとじゃんけんをしましょう。
`INT(RND * 3)` は 0, 1, 2 のどれかをランダムに作る定石です
(くわしくは第 11 章で):

```basic
' janken.bas — じゃんけん
RANDOMIZE                      ' 乱数のサイコロを毎回振り直す
PRINT "じゃんけん! (0=グー 1=チョキ 2=パー)"
INPUT "あなたの手"; ME%
COM% = INT(RND * 3)
PRINT "コンピュータ:"; COM%
IF ME% = COM% THEN
  PRINT "あいこ!"
ELSEIF (ME% = 0 AND COM% = 1) OR (ME% = 1 AND COM% = 2) OR (ME% = 2 AND COM% = 0) THEN
  PRINT "あなたの勝ち!"
ELSE
  PRINT "あなたの負け…"
END IF
```

勝ち判定の長い条件式を声に出して読んでみてください。
「グーでチョキに勝つ、または、チョキでパーに勝つ、または…」。
条件式は日本語に訳せるように書くのがコツです。

## 他の言語では

```text
NBASIC-21 :  IF X >= 60 THEN ... ELSE ... END IF
Python    :  if x >= 60: ... else: ...
JavaScript:  if (x >= 60) { ... } else { ... }
C         :  if (x >= 60) { ... } else { ... }
```

`SELECT CASE` も `switch` という名前でたいていの言語にあります。
「条件で道が分かれる」という構造は 100% 共通です。

## 練習問題

1. 数を 1 つ入力させて、「正の数」「負の数」「ゼロ」のどれかを
   表示するプログラムを書いてください。
2. 身長 (m) と体重 (kg) を聞いて BMI (体重 ÷ 身長の 2 乗) を計算し、
   18.5 未満なら「やせ型」、25 未満なら「ふつう」、それ以上なら
   「がっちり」と表示するプログラムを書いてください。
3. 西暦を聞いて、うるう年かどうかを判定するプログラムを書いて
   ください。ルール: 「4 で割り切れたらうるう年。ただし 100 で
   割り切れたら平年。ただし 400 で割り切れたらやっぱりうるう年」。
   (ヒント: `MOD` と `AND`/`OR` の出番です。2000 年はうるう年、
   1900 年は平年になれば正解。)
