# 第 8 章 部品に分ける — SUB と FUNCTION

プログラムが 50 行を超えたあたりから、全体を一度に頭に入れるのが
つらくなってきます。プロはどうしているのか? —
**大きな仕事を、名前のついた小さな部品に分ける**のです。
この章はプログラミング上達の最大の分岐点です。

## 8.1 SUB — 名前をつけた手順書

同じ処理を 3 か所で使いたいとき、3 回コピペする代わりに、
処理に**名前をつけて**呼び出せます。それが `SUB` (サブルーチン) です:

```basic
' メインプログラム
DrawLine
PRINT "お知らせ"
DrawLine
PRINT "本日は晴天なり"
DrawLine

' ---- 部品置き場 ----
SUB DrawLine
  PRINT "----------------------------------------"
END SUB
```

```text
----------------------------------------
お知らせ
----------------------------------------
本日は晴天なり
----------------------------------------
```

`SUB 名前 〜 END SUB` で部品を定義し、名前を書くだけで呼び出せます。
呼び出すと部品の中身が実行され、終わると**呼び出した場所の続き**に
戻ってきます。部品はプログラムのどこに置いても構いません
(実行は上から順ですが、SUB の中身は「呼ばれたときだけ」動きます)。

## 8.2 引数 — 部品に材料を渡す

部品に**材料** (引数) を渡すと、ぐっと便利になります:

```basic
Greet "太郎", 3
Greet "花子", 1

SUB Greet (NAME$, TIMES%)
  FOR I% = 1 TO TIMES%
    PRINT "こんにちは、"; NAME$; "さん!"
  NEXT I%
END SUB
```

```text
こんにちは、太郎さん!
こんにちは、太郎さん!
こんにちは、太郎さん!
こんにちは、花子さん!
```

`SUB Greet (NAME$, TIMES%)` の NAME$ と TIMES% を**仮引数**と
いいます。呼び出し時の値 (`"太郎", 3`) がそこにセットされて
部品が動きます。同じ部品が、渡す材料しだいで違う仕事をする —
これが部品化の威力です。

## 8.3 FUNCTION — 答えを返す部品

`SUB` はやりっぱなしの部品でしたが、`FUNCTION` は**答え (戻り値) を
返す**部品です。`LEN()` や `INT()` を自分で作れると思ってください:

```basic
PRINT Bmi(1.7, 60)
PRINT Bmi(1.6, 70)

FUNCTION Bmi (HEIGHT, WEIGHT)
  Bmi = WEIGHT / (HEIGHT * HEIGHT)   ' 関数名に代入 = これが答え
END FUNCTION
```

```text
 20.7612456747405 
 27.34375 
```

FUNCTION の中で**関数名そのものに代入**したものが戻り値になります。
`RETURN 式` と書いて即座に返すこともできます (現代風):

```basic
PRINT Max2%(3, 8); Max2%(10, 2)

FUNCTION Max2% (A%, B%)
  IF A% > B% THEN RETURN A%
  RETURN B%
END FUNCTION
```

名前に `%` や `$` を付ければ、返す値の型もそれで決まります。
「2 つのうち大きい方」のような小さな道具を FUNCTION にしておくと、
本体のプログラムが日本語のように読めるようになります。

## 8.4 変数の見える範囲 — スコープ

重要ルール: **部品の中の変数は、部品の外とは別物**です。

```basic
X% = 100
Test
PRINT "メインの X% は"; X%      ' 100 のまま!

SUB Test
  X% = 999                       ' これは Test 専用の X%
END SUB
```

```text
メインの X% は 100 
```

部品の中の `X%` は、名前が同じでも**まったく別の箱**
(ローカル変数といいます)。だから部品を作るとき、メイン側で
どんな変数名が使われているか気にしなくてよいのです。
部品が増えても混線しない — これがスコープという仕組みの
ありがたみです。

どうしても部品とメインで共有したい変数は、メインの先頭で
`DIM` して作ります (共有変数・グローバル変数):

```basic
DIM TOTAL AS DOUBLE       ' 先頭で DIM した変数は部品からも見える

AddFee 1000
AddFee 2500
PRINT "合計:"; TOTAL

SUB AddFee (V)
  TOTAL = TOTAL + V       ' 共有の TOTAL に足し込む
END SUB
```

ただし共有変数は「どこから書き換えられたか分からなくなる」
副作用があるので、**基本は引数と戻り値、共有はどうしてもの
ときだけ**が良い設計です。これはあらゆる言語共通の鉄則です。

## 8.5 再帰 — 自分を呼ぶ関数

FUNCTION は自分自身を呼べます。**再帰**と呼ばれる、強力だけど
最初は頭がくらくらするテクニックです。階乗 (N! = 1×2×…×N) で
見てみましょう。階乗にはこういう性質があります:

> 5! = 5 × 4!   つまり「N の階乗は、N × (N-1) の階乗」

「階乗を求めるのに階乗を使う」— これをそのまま書きます:

```basic
PRINT Fact#(5)     ' 120
PRINT Fact#(10)    ' 3628800

FUNCTION Fact# (N%)
  IF N% <= 1 THEN
    Fact# = 1                    ' (1) 一番小さい場合の答えは直接書く
  ELSE
    Fact# = N% * Fact#(N% - 1)   ' (2) あとは「一回り小さい自分」に任せる
  END IF
END FUNCTION
```

再帰には必ず 2 つの部品があります:

1. **底** (基底ケース) — これ以上小さくならない場合の直接の答え。
   `N% <= 1 なら 1`。これを忘れると無限に自分を呼んで暴走します。
2. **一歩** — 問題を一回り小さくして自分に任せる。`N% × Fact#(N%-1)`。

`Fact#(3)` の動きを追うと: 3×Fact#(2) → 3×(2×Fact#(1)) → 3×(2×1) = 6。
「任せて、任せて、底に着いたら答えが折り返してくる」イメージです。

いまは「こういう書き方がある」と知るだけで十分です。再帰は
第 10 章 (クイックソート) と第 15 章で再登場します。

## 8.6 実践: プログラムを部品で組み立てる

部品化の実感がわく例として、「成績表プログラム」を部品で
組み立てます。**メイン部分だけ読めば何をするか分かる**ことに
注目してください:

```basic
' seiseki.bas — 部品で組み立てた成績集計
DIM SCORE%(4)
FOR I% = 0 TO 4
  READ SCORE%(I%)
NEXT I%
DATA 80, 65, 92, 71, 58

PrintHeader "成績レポート"
PRINT "平均点  :"; Average(SCORE%(0), SCORE%(1), SCORE%(2), SCORE%(3), SCORE%(4))
PRINT "最高評価: "; Grade$(92)
PRINT "最低評価: "; Grade$(58)

SUB PrintHeader (TITLE$)
  PRINT "========================"
  PRINT " "; TITLE$
  PRINT "========================"
END SUB

FUNCTION Average (A%, B%, C%, D%, E%)
  Average = (A% + B% + C% + D% + E%) / 5
END FUNCTION

FUNCTION Grade$ (S%)
  IF S% >= 80 THEN
    Grade$ = "A"
  ELSEIF S% >= 60 THEN
    Grade$ = "B"
  ELSE
    Grade$ = "C"
  END IF
END FUNCTION
```

> **コラム: GOSUB — ご先祖さまのサブルーチン**
> 古い BASIC には SUB がなく、`GOSUB 行番号` で「その行に飛んで、
> RETURN で戻る」というやり方をしていました。NBASIC-21 でも
> 動きます (第 1 章の examples/classic.bas 参照) が、引数もローカル
> 変数もないため大きなプログラムでは混乱のもとでした。SUB/FUNCTION
> は、その反省から生まれた道具です。

## 他の言語では

```text
NBASIC-21 :  FUNCTION Bmi (H, W) ... END FUNCTION
Python    :  def bmi(h, w): return w / (h * h)
JavaScript:  function bmi(h, w) { return w / (h * h); }
C         :  double bmi(double h, double w) { return w / (h * h); }
```

名前は違えど (関数、メソッド、プロシージャ…)、
「引数を受け取り、値を返す部品」はプログラミングの世界共通語です。
スコープ (ローカル変数) の考え方もそのまま通用します。

## 練習問題

1. 西暦を受け取ってうるう年なら -1 (真)、平年なら 0 を返す
   `FUNCTION IsLeap%(Y%)` を書き、1996〜2004 年を FOR ループで
   判定させてください (第 4 章の練習 3 を部品化するわけです)。
2. `FUNCTION Min2%(A%, B%)` (2 つの小さい方) を書き、
   それを使って `FUNCTION Min3%(A%, B%, C%)` (3 つの最小)
   を書いてください。部品が部品を使う練習です。
3. 文字列 S$ を N 回くり返した文字列を返す
   `FUNCTION Repeat$(S$, N%)` を書いてください
   (`Repeat$("AB", 3)` → `"ABABAB"`)。
4. (再帰に挑戦) `FUNCTION Fib%(N%)` — フィボナッチ数列
   (1, 1, 2, 3, 5, 8, … 前の 2 つの和) の N 番目を再帰で
   求めてください。底は「N が 2 以下なら 1」です。
