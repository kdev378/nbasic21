# 第 5 章 くり返す — ループ

コンピュータの最大の特技は「同じことを、文句も言わず、超高速で、
何万回でもくり返す」ことです。その力を引き出すのが**ループ**です。
この章を終えると、あなたのプログラムは人間には不可能な仕事を
こなしはじめます。

## 5.1 FOR — 回数が決まっているくり返し

「1 から 5 まで数えながら 5 回くり返す」はこう書きます:

```basic
FOR I% = 1 TO 5
  PRINT I%; "回目のこんにちは"
NEXT I%
PRINT "おわり"
```

```text
 1 回目のこんにちは
 2 回目のこんにちは
 3 回目のこんにちは
 4 回目のこんにちは
 5 回目のこんにちは
おわり
```

動きを追いましょう。`FOR I% = 1 TO 5` は「箱 I% に 1 を入れて
スタート。`NEXT` まで来たら I% に 1 足して戻る。I% が 5 を超えたら
くり返し終了」という意味です。I% を**ループカウンタ**と呼びます。
カウンタが「今何回目か」を常に教えてくれるのがミソです。

`STEP` を付けると増え方を変えられます:

```basic
FOR I% = 10 TO 0 STEP -2      ' 10 から 0 まで 2 ずつ減る
  PRINT I%;
NEXT I%
PRINT
```

```text
 10  8  6  4  2  0 
```

## 5.2 累積 — ループ最大の必殺技

ループと変数を組み合わせた**累積**は、プログラミング全体で
もっとも重要なパターンです。「1 から 100 までの合計」を求めます:

```basic
TOTAL% = 0                 ' 合計の箱を 0 で用意しておく
FOR I% = 1 TO 100
  TOTAL% = TOTAL% + I%     ' 今の合計に I% を足し込む
NEXT I%
PRINT "1〜100 の合計は"; TOTAL%
```

```text
1〜100 の合計は 5050 
```

ポイントは 2 つ。**ループの前に箱を 0 にしておく**こと、
**ループの中で足し込む**こと。この型は合計だけでなく、
「個数を数える」「最大値を探す」「文字をつなげる」など、
姿を変えて何度でも登場します。

## 5.3 ループの中に分かれ道

ループと IF は自由に組み合わせられます。1〜30 のうち
3 の倍数だけ表示してみます:

```basic
FOR I% = 1 TO 30
  IF I% MOD 3 = 0 THEN
    PRINT I%;
  END IF
NEXT I%
PRINT
```

```text
 3  6  9  12  15  18  21  24  27  30 
```

有名な **FizzBuzz** もこの組み合わせだけで書けます。
「3 の倍数なら Fizz、5 の倍数なら Buzz、両方の倍数なら FizzBuzz、
それ以外は数そのまま」:

```basic
FOR N% = 1 TO 20
  IF N% MOD 15 = 0 THEN
    PRINT "FizzBuzz"
  ELSEIF N% MOD 3 = 0 THEN
    PRINT "Fizz"
  ELSEIF N% MOD 5 = 0 THEN
    PRINT "Buzz"
  ELSE
    PRINT N%
  END IF
NEXT N%
```

なぜ 15 の判定が最初なのか? — 15 の倍数は 3 の倍数でもあるので、
3 を先に調べると Fizz で終わってしまうからです (第 4 章の
「上から最初のひとつ」を思い出してください)。FizzBuzz は
プログラマの採用面接で本当に使われる問題です。もう解けましたね。

## 5.4 二重ループ — ループの中にループ

九九の表を作ります:

```basic
FOR I% = 1 TO 9
  FOR J% = 1 TO 9
    PRINT I% * J%;
  NEXT J%
  PRINT                    ' 1 段ごとに改行
NEXT I%
```

外側の I% が 1 のあいだに、内側の J% が 1〜9 を一周します。
つまり中身は 9 × 9 = 81 回動きます。「外が 1 目盛り進むごとに、
内側が一周する」— 時計の長針と短針の関係です。

## 5.5 WHILE — 条件が成り立つあいだくり返す

回数が事前にわからないくり返しは `WHILE` を使います。
「条件が成り立っているあいだ、回り続ける」:

```basic
' 貯金が 100 万円を超えるのは何年後? (年利 5% の複利)
MONEY = 500000
YEARS% = 0
WHILE MONEY < 1000000
  MONEY = MONEY * 1.05
  YEARS% = YEARS% + 1
WEND
PRINT YEARS%; "年後 ("; INT(MONEY); "円)"
```

```text
 15 年後 ( 1039464 円)
```

`WHILE 条件` 〜 `WEND` の形です。条件はループの**入り口で毎回**
チェックされます。最初から成り立っていなければ 1 回も実行されません。

## 5.6 DO LOOP — 「最低 1 回」と「無限ループ」

`DO 〜 LOOP` は WHILE の親戚で、条件を**後ろ**に書けます。
後ろに書くと「まずやってから判定」なので、最低 1 回は実行されます。
入力チェックの定番です:

```basic
DO
  INPUT "1〜10 の数をどうぞ"; N%
LOOP UNTIL N% >= 1 AND N% <= 10
PRINT N%; "を受け付けました"
```

`UNTIL` は「〜になるまで」。正しい数が来るまで何度でも聞き直します。

条件を書かないと**無限ループ**になります。抜けるには `EXIT DO`:

```basic
' 合計いくつ? 0 で終了
TOTAL% = 0
DO
  INPUT "数 (0 で終了)"; N%
  IF N% = 0 THEN EXIT DO
  TOTAL% = TOTAL% + N%
LOOP
PRINT "合計:"; TOTAL%
```

`EXIT FOR`・`EXIT WHILE` もあり、それぞれのループから途中で
脱出できます。

> **暴走したら**: 間違えて終わらない無限ループを作ってしまったら、
> ターミナルで `Ctrl + C` を押せば強制終了できます。誰でも一度は
> やります。通過儀礼です。

## 5.7 実践: 数当てゲーム

ここまでの武器 (変数・IF・ループ・乱数) を全部使うと、
遊べるゲームが作れます:

```basic
' kazuate.bas — 数当てゲーム
RANDOMIZE
SECRET% = INT(RND * 100) + 1   ' 1〜100 の秘密の数
TRIES% = 0
PRINT "1〜100 の数を当ててください"
DO
  INPUT "あなたの予想"; G%
  TRIES% = TRIES% + 1
  IF G% < SECRET% THEN
    PRINT "もっと大きい"
  ELSEIF G% > SECRET% THEN
    PRINT "もっと小さい"
  ELSE
    PRINT TRIES%; "回で正解!"
    EXIT DO
  END IF
LOOP
```

遊んでみてください。そして考えてみてください —
「もっとも少ない回数で当てる作戦」は何でしょう?
答えは第 9 章 (探索アルゴリズム) で明かされます。実は
このゲームの必勝法は、コンピュータ科学の重要アルゴリズム
そのものなのです。

## 他の言語では

```text
NBASIC-21 :  FOR I% = 1 TO 5 ... NEXT I%
Python    :  for i in range(1, 6): ...
JavaScript:  for (let i = 1; i <= 5; i++) { ... }
C         :  for (int i = 1; i <= 5; i++) { ... }
```

WHILE はほぼ全言語に同じ名前・同じ意味であります。
「回数が決まっているなら FOR、決まっていないなら WHILE」という
使い分けも共通の感覚です。

   ```text
   *
   **
   ***
   ****
   *****
   ```

   表示し、7 回以内に当てられなかったら負け、にしてください。
