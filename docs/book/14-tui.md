# 第 14 章 画面を自由に描く — TUI 入門

いままでの出力は「上から下に流れる」だけでした。この章では
ターミナルの画面を**方眼紙**として自由に使います。好きな場所に
文字を置き、色を塗り、キー入力に即座に反応する — いわゆる
**TUI** (Text User Interface) です。次章のゲーム作りの
基礎訓練でもあります。

## 14.1 新しい道具 5 つ

| 命令/関数 | 意味 |
|---|---|
| `CLS` | 画面を消してカーソルを左上へ |
| `LOCATE 行, 桁` | カーソルを指定位置へ (左上が 1, 1) |
| `COLOR 文字色 [, 背景色]` | 以後の文字の色を変える |
| `SLEEP 秒` | 指定秒だけ待つ (0.1 のような小数も可) |
| `INKEY$` | 押されたキーを 1 つ返す。**押されていなければ待たずに ""** |

まずは位置指定から。画面のど真ん中あたりに文字を置いてみます:

```basic
CLS
LOCATE 12, 35
PRINT "まんなか!"
LOCATE 20, 1          ' 最後にカーソルを下へ逃がしておく (お作法)
```

`LOCATE 行, 桁` は「上から 12 行目、左から 35 桁目」。
座標が (行, 桁) の順で、**縦が先**なことに注意してください
(数学の (x, y) とは逆です)。

## 14.2 色を塗る

色は番号で指定します。伝統の 16 色パレットです:

| 番号 | 色 | 番号 | 色 |
|---|---|---|---|
| 0 | 黒 | 8 | 灰色 |
| 1 | 青 | 9 | 明るい青 |
| 2 | 緑 | 10 | 明るい緑 |
| 3 | 水色 | 11 | 明るい水色 |
| 4 | 赤 | 12 | 明るい赤 |
| 5 | 紫 | 13 | 明るい紫 |
| 6 | 黄土 | 14 | 黄色 |
| 7 | 白 | 15 | 明るい白 |

```basic
' colorbar.bas — 16 色見本
CLS
FOR C% = 0 TO 15
  COLOR C%
  PRINT "色番号"; C%; "のサンプル ■■■■"
NEXT C%
COLOR 7              ' 白に戻しておく (お作法)
```

`COLOR 14, 1` のように 2 つ目を書くと背景色になります。
プログラムの最後で `COLOR 7` に戻すのを忘れると、終了後の
ターミナルが変な色のままになります (壊れてはいないので、
ターミナルを開き直せば直ります)。

## 14.3 動かす — アニメーションの原理

アニメーションの原理は「**消して、少しずらして、描く**」の
くり返しです。ボールが画面を横切ります:

```basic
' ball.bas — 転がるボール
CLS
FOR X% = 1 TO 60
  LOCATE 10, X%
  PRINT " O";         ' 先頭の空白が「前の位置の O」を消す
  SLEEP 0.03
NEXT X%
LOCATE 20, 1
```

`" O"` の空白が芸です。新しい位置に O を描くと同時に、
1 桁左 (= さっきまで O がいた場所) を空白で上書きしています。
`SLEEP 0.03` は 1 コマ 0.03 秒 — 秒 30 コマのパラパラ漫画です。

## 14.4 INKEY$ — 待たない入力

`INPUT` は Enter が押されるまでプログラムを**止めて**しまいます。
ゲームでは困ります (敵は待ってくれない)。`INKEY$` は
「押されていればそのキー、いなければ空文字列を**即座に**返す」
関数です。

これを無限ループと組み合わせたものが、あらゆるゲームの心臓部
**ゲームループ**です:

```basic
' gameloop.bas — ゲームループの骨格 (q で終了)
CLS
X% = 40                        ' 自機の位置
DO
  ' ---- (1) 入力を調べる (待たない!) ----
  K$ = INKEY$
  IF K$ = "q" THEN EXIT DO
  IF K$ = "a" AND X% > 1 THEN X% = X% - 1     ' 左へ
  IF K$ = "d" AND X% < 78 THEN X% = X% + 1    ' 右へ

  ' ---- (2) 描く ----
  LOCATE 12, 1
  PRINT SPACE$(79);            ' 12 行目をまるごと消して
  LOCATE 12, X%
  PRINT "@";                   ' 自機を描く
  LOCATE 1, 1
  PRINT "a=左 d=右 q=終了  位置:"; X%; "  ";

  ' ---- (3) ひと呼吸 ----
  SLEEP 0.03
LOOP
LOCATE 20, 1
PRINT "おつかれさま!"
```

「入力 → 更新 → 描画 → 待つ」を毎秒 30 回ほど回す。
この骨格は、スーパーマリオも最新の 3D ゲームも同じです。
`X% > 1` のような**範囲チェック**で、自機が画面外へ出ないように
している点にも注目 (画面外への LOCATE はエラーになります)。

`SPACE$(N)` は空白 N 個の文字列を作る関数で、行の消去に便利です。

## 14.5 実践: 反射神経ゲーム

この章の道具だけで、遊べるゲームがもう作れます:

```basic
' reflex.bas — 反射神経テスト
CLS
PRINT "画面に ! が出たら、できるだけ速く何かキーを押して!"
PRINT "(押すのが早すぎたら失格)"
SLEEP 2

RANDOMIZE
WAITTIME = 2 + RND * 3         ' 2〜5 秒のランダムな待ち

' フライング検出しながら待つ
T0 = TIMER
DO WHILE TIMER - T0 < WAITTIME
  IF INKEY$ <> "" THEN
    PRINT "フライング! 失格です"
    END
  END IF
LOOP

CLS
LOCATE 12, 40
COLOR 12
PRINT "!"
COLOR 7
T0 = TIMER
DO
  K$ = INKEY$
LOOP UNTIL K$ <> ""            ' 何か押されるまで回る
LOCATE 14, 30
PRINT "反応時間:"; TIMER - T0; "秒"
LOCATE 20, 1
```

`TIMER` は「深夜 0 時からの経過秒数」を返す関数で、
2 回測って引き算すれば経過時間になります (ストップウォッチの
原理)。精度は 1 秒単位なので、このゲームでは「1 秒未満 = 表示 0」
になることもあります。ご愛嬌ということで — 気になる人は
「何回ループを回れたか」で計る方式に改造してみてください
(練習問題 4)。

## 14.6 TUI プログラムの心得

- **最後は後片付け**: `COLOR 7` で色を戻し、`LOCATE` でカーソルを
  画面下へ。散らかしたら片付ける、です。
- **画面外に描かない**: LOCATE の行・桁が 1 未満だと実行時エラー。
  座標を動かす前に範囲チェックを。
- **点滅がひどいときは**: 画面全体を CLS で消して全部描き直すと
  ちらつきます。14.4 のように「変わった部分だけ」消して描くのが
  滑らかさのコツです。

## 他の言語では

ターミナルの画面制御は、実はどの言語でも「エスケープ
シーケンス」という共通の仕組みの上に載っています (Python の
curses、Rust の crossterm など)。`LOCATE 12, 35` の裏では
`ESC[12;35H` という暗号のような文字列が端末に送られています —
NBASIC-21 がそれを隠して簡単にしてくれているだけです。
「入力・更新・描画のゲームループ」は Unity でもブラウザゲームでも
同じ構造です。

## 練習問題

1. 自分の名前を、画面のランダムな位置にランダムな色で 20 個
   表示するプログラムを書いてください。
2. ボールのアニメーション (14.3) を「右端に着いたら跳ね返って
   左へ戻る」ように改造してください (ヒント: 進む向きを変数に)。
3. ゲームループ (14.4) に w / s キーで上下移動も追加して、
   画面中を自由に動き回れるようにしてください。
4. 反射神経ゲームを「! が出てからキーが押されるまでにループが
   何回回ったか」で計るように改造してください。数字が小さいほど
   速い、という得点になります。
