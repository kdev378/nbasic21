# 第 1 章 道具をそろえる

この章のゴールはただひとつ。**あなたのパソコンで、プログラムを 1 本
動かすこと**です。ここが一番つまずきやすい場所なので、ゆっくり丁寧に
いきます。うまくいかなくても、あなたのせいではありません。手順どおりに
落ち着いて進めましょう。

## 1.1 これから何をするのか

プログラミングとは「コンピュータへの命令書を書くこと」です。
命令書はただのテキスト (文字) ですが、コンピュータが直接理解できるのは
機械語という数字の羅列だけです。そこで、

1. 人間が読み書きできる言葉 (NBASIC-21) で命令書を書く
2. **コンパイラ**という翻訳ソフトで機械語に変換する
3. できあがった実行ファイルを動かす

という 3 ステップを踏みます。この本ではステップ 2 と 3 を
`nbc run` というひとつのコマンドにまとめてあるので、
実際にやることは「書く」と「nbc run」の 2 つだけです。

そのために必要な道具は 3 つ:

| 道具 | 役割 |
|---|---|
| **Python** | NBASIC-21 コンパイラを動かすためのソフト |
| **C コンパイラ (gcc)** | 機械語への最終変換を担当するソフト |
| **テキストエディタ** | 命令書を書くソフト。**VS Code** をおすすめします |

## 1.2 ターミナルを開く

これからの作業は「ターミナル」という黒い画面で行います。
文字でコンピュータに指示を出すための窓です。

- **Windows**: スタートボタンを押して `cmd` と打ち、
  「コマンドプロンプト」をクリック。
- **Linux (Ubuntu など)**: `Ctrl + Alt + T`、またはアプリ一覧から
  「端末」(Terminal)。

黒い画面に `>` や `$` で終わる 1 行が表示されていれば成功です。
これは「入力待ってます」の合図で、**プロンプト**と呼びます。

> **こわがらなくて大丈夫**
> ターミナルで何かを打ち間違えても、パソコンは壊れません。
> 変なことになったら窓を閉じて開き直せば元通りです。

## 1.3 道具のインストール

### Windows の場合

1. **Python** — [python.org/downloads](https://www.python.org/downloads/)
   から「Download Python 3.x.x」の黄色いボタンでインストーラを入手し、
   実行します。最初の画面で **「Add python.exe to PATH」に必ずチェック**
   を入れてから「Install Now」を押してください (ここが最重要ポイント)。
2. **gcc** — [winlibs.com](https://winlibs.com/) から
   「UCRT runtime」の「Win64」の Zip をダウンロードし、
   `C:\mingw64` に展開します。その後、
   スタートで「環境変数」と検索 → 「環境変数を編集」→ 「Path」を選んで
   「編集」→ 「新規」で `C:\mingw64\bin` を追加して OK。
3. **このリポジトリ** — [リポジトリのページ](https://github.com/kdev378/nbasic21)
   の緑の「Code」ボタン → 「Download ZIP」でダウンロードし、
   `C:\nbasic21` など分かりやすい場所に展開します。

確認します。コマンドプロンプトを**開き直して** (ここも重要)、
次の 2 つを打ってみてください (1 行打つたびに Enter):

```text
py --version
gcc --version
```

それぞれ `Python 3.…`、`gcc …` のようにバージョンが表示されれば
準備完了です。「認識されていません」と出たら、この節の手順の
チェック項目 (PATH) を見直してください。

### Linux (Ubuntu / Debian 系) の場合

ターミナルで次の 2 行を打つだけです (`sudo` はパスワードを聞かれます):

```text
sudo apt update
sudo apt install -y python3 gcc git
git clone https://github.com/kdev378/nbasic21.git
```

`python3 --version` と `gcc --version` でバージョンが出れば完了です。

### VS Code (どちらの OS でも)

[code.visualstudio.com](https://code.visualstudio.com/) からインストール
してください。そして NBASIC-21 用の拡張機能を入れます:

- **Windows**: コマンドプロンプトで
  `xcopy /e /i C:\nbasic21\editors\vscode %USERPROFILE%\.vscode\extensions\nbasic21.nbasic21-0.1.0`
- **Linux**:
  `cp -r ~/nbasic21/editors/vscode ~/.vscode/extensions/nbasic21.nbasic21-0.1.0`

VS Code を起動し直すと、`.bas` ファイルが色付きで表示され、
**F5 キーだけでプログラムを実行**できるようになります。

## 1.4 はじめてのプログラム

いよいよです。まず作業用のフォルダを作ります。ターミナルで
リポジトリのフォルダに移動して (`cd` は「フォルダを移動する」命令):

```text
cd C:\nbasic21        ← Windows の場合
cd ~/nbasic21         ← Linux の場合
mkdir work
```

VS Code で「ファイル → フォルダーを開く」からこの `nbasic21` フォルダを
開き、`work` フォルダの中に `hello.bas` という名前で新しいファイルを
作って、次の 2 行を打ち込んでください:

```basic
PRINT "こんにちは、コンピュータ!"
PRINT 1 + 2 + 3
```

保存 (`Ctrl + S`) したら、ターミナルで:

```text
tools\nbc run work\hello.bas      ← Windows
tools/nbc run work/hello.bas      ← Linux
```

こう表示されたら — おめでとうございます。あなたは今、
プログラマーになりました。

```text
こんにちは、コンピュータ!
 6
```

VS Code の拡張を入れた人は、`hello.bas` を開いて **F5 を押すだけ**でも
同じことができます。以降の章では `nbc run` の形で書きますが、
F5 と読み替えてかまいません。

## 1.5 何が起きたのか

`nbc run` の裏側では、こういう流れが動いています:

```text
hello.bas ──(NBASIC-21 コンパイラ)──> hello.c ──(gcc)──> hello (実行ファイル) ──> 実行
```

つまりあなたの書いた命令書は、いったん C 言語という別の言語に翻訳され、
さらに機械語の実行ファイルに変換されてから動いています。
できあがった実行ファイル (`work` フォルダの `hello` または `hello.exe`)
は単体で動くので、コンパイラのないパソコンに持っていっても動きます。
これが「コンパイル方式」の言語の特徴です。

## 1.6 つまずいたら

| 症状 | 対処 |
|---|---|
| `'py' は認識されていません` (Windows) | Python のインストールで「Add python.exe to PATH」を忘れた可能性。インストーラをもう一度実行し「Modify」から直せます |
| `'gcc' は認識されていません` | PATH の追加 (1.3 節) を確認し、**ターミナルを開き直す** |
| `nbc: Python が見つかりません` | 上と同じ。ターミナルの開き直しを忘れずに |
| `そのようなファイルやディレクトリはありません` | 今いるフォルダが違います。`cd` でリポジトリのフォルダに移動したか確認 |
| 赤い文字で `error:` が出る | プログラムの打ち間違いです。第 2 章で読み方を学びます |

## 練習問題

1. `hello.bas` の 1 行目のメッセージを、自分の好きな言葉に変えて
   実行し直してみましょう。
2. `PRINT 1 + 2 + 3` の下に、`PRINT 365 * 24` という行を足して、
   1 年が何時間なのか計算させてみましょう。

答え合わせは不要です — 動けば正解。それがプログラミングです。
