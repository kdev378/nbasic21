# 第 2 章 コンピュータに命令する

この章では、いちばん基本の命令 `PRINT` と、計算のやり方、そして
**エラーが出たときの読み方**を学びます。地味に見えますが、
この章の内容はこの先ずっと、毎日使います。

## 2.1 PRINT — 画面に表示する

`PRINT` は「画面に表示しろ」という命令です。前の章で既に使いましたね。

```basic
PRINT "おはよう"
PRINT "世界"
```

```text
おはよう
世界
```

ルールをまとめます:

- **1 行に 1 つの命令**を書く。上から順に実行される。
- 文字をそのまま表示したいときは `"` (ダブルクォート) で囲む。
  この囲まれたかたまりを**文字列**と呼びます。
- `"` で囲まなかったものは**計算式**として扱われます。

その違いを見てみましょう:

```basic
PRINT "1 + 1"
PRINT 1 + 1
```

```text
1 + 1
 2
```

1 行目は文字列なのでそのまま、2 行目は式なので計算されて `2` に
なりました。数字の前に半角スペースが入るのは BASIC の伝統的な作法で、
マイナスの数のときにここへ `-` が入ります。

## 2.2 電卓として使う

PRINT と計算式を組み合わせれば、もう電卓です。使える記号 (演算子と
いいます) はこれだけあります:

| 記号 | 意味 | 例 | 結果 |
|---|---|---|---|
| `+` | たし算 | `PRINT 7 + 3` | 10 |
| `-` | ひき算 | `PRINT 7 - 3` | 4 |
| `*` | かけ算 | `PRINT 7 * 3` | 21 |
| `/` | わり算 | `PRINT 7 / 3` | 2.33333… |
| `\` | わり算の商 (整数) | `PRINT 7 \ 3` | 2 |
| `MOD` | わり算の余り | `PRINT 7 MOD 3` | 1 |
| `^` | べき乗 (○の△乗) | `PRINT 2 ^ 10` | 1024 |

かけ算が `×` でなく `*`、わり算が `÷` でなく `/` なのは、
キーボードにある記号を使うためです。世界中のほぼすべての
プログラミング言語がこの記号を使います。

`\` と `MOD` のペアは「7 個のあめを 3 人で分けると、1 人 2 個で
1 個余る」の「2」と「1」です。地味ですが、この先おどろくほど
よく使います (偶数・奇数の判定、時間の計算、など)。

計算の順序は算数と同じです。かけ算・わり算が先、かっこが最優先:

```basic
PRINT 2 + 3 * 4        ' 14 (3*4 が先)
PRINT (2 + 3) * 4      ' 20 (かっこが先)
```

## 2.3 コメント — 人間のためのメモ

上の例に出てきた `'` から行末までは**コメント**といって、
コンパイラに完全に無視されます。つまり人間のためのメモ欄です。

```basic
' このプログラムは消費税を計算します (この行は無視される)
PRINT 1980 * 1.1       ' 税込み価格 (行の途中からでも書ける)
```

「そんなの要る?」と思うかもしれませんが、1 週間後のあなたは
他人です。未来の自分への手紙だと思って書く習慣をつけましょう。
古い BASIC との互換で `REM` という書き方もありますが、`'` が楽です。

## 2.4 1 行にまとめて表示する

`PRINT` にカンマやセミコロンで区切って複数の項目を渡せます:

```basic
PRINT "たて 3cm よこ 5cm の長方形"
PRINT "面積は"; 3 * 5; "平方センチ"
PRINT "A", "B", "C"
```

```text
たて 3cm よこ 5cm の長方形
面積は 15 平方センチ
A             B             C
```

- `;` (セミコロン) — **くっつけて**表示する
- `,` (カンマ) — **表の列のように**位置をそろえて表示する

行末に `;` を置くと「改行しないで次の PRINT に続ける」という意味に
なります。これは後の章で活躍します。

## 2.5 エラーと友だちになる

わざとエラーを起こしてみましょう。次のプログラムには間違いがあります:

```basic
' (これは断片です — わざと間違えている例)
PRINT "こんにちは
```

実行すると、こう表示されます:

```text
work/hello.bas:2:7: error: 文字列リテラルが閉じていません
```

読み方はこうです:

```text
work/hello.bas : 2 : 7 :  error: 文字列リテラルが閉じていません
└ どのファイル   └行  └桁   └ 何が起きたか
```

「hello.bas の 2 行目、7 文字目あたりで、`"` で始めた文字列が
閉じられていない」と教えてくれています。実際、行末の `"` を
忘れていますね。**エラーは叱られているのではなく、場所つきの
ヒントをもらっている**のです。プロのプログラマも 1 日に何十回も
エラーを出します。違いは、エラー文をちゃんと読むかどうかだけです。

よくあるエラーをまとめました。困ったら
[付録 B: エラーメッセージ辞典](appendix-b-errors.md) を見てください。

| エラー | よくある原因 |
|---|---|
| 文字列リテラルが閉じていません | `"` の閉じ忘れ |
| 式が必要ですが 行末 があります | `PRINT 1 +` のように式が途中で終わっている |
| 不正な文字です | 全角スペースや全角記号が紛れ込んだ (※) |

※ **全角には要注意。** 日本語入力のまま ` `(全角スペース) や
`＋`(全角プラス) を打ってしまうのは、日本の初心者の
つまずき原因ナンバーワンです。記号と数字は必ず半角で。
文字列の中 (`"..."` の中) とコメントだけは日本語 OK です。

## 2.6 プログラムは「上から順に」

最後に、いちばん大事な原則をひとつ。

> プログラムは、**上の行から順に、1 行ずつ**実行される。

```basic
PRINT "3"
PRINT "2"
PRINT "1"
PRINT "発射!"
```

```text
3
2
1
発射!
```

当たり前に見えますか? でもこの「順番どおり」こそがプログラムの
本質です。この先、順番を**曲げる**方法 (分岐、ループ) を学びますが、
それらはすべて「基本は上から順」の上に成り立っています。

## 他の言語では

画面表示の命令はどの言語にもあります。考え方は全部同じです:

```text
NBASIC-21 :  PRINT "Hello"
Python    :  print("Hello")
JavaScript:  console.log("Hello")
C         :  printf("Hello\n");
```
