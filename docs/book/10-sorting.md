# 第 10 章 並べ替える — ソートアルゴリズム

前章の二分探索は「並んでいるデータ」が前提でした。では、どうやって
並べるのか? **ソート** (並べ替え) は、アルゴリズムの教科書で必ず
主役を張る古典中の古典です。素朴なものから 3 つ、じっくり
手を動かして身につけましょう。

## 10.1 準備: 交換 (SWAP)

並べ替えの基本動作は「2 つの箱の中身を入れ替える」ことです。
NBASIC-21 には専用命令 `SWAP` があります:

```basic
A% = 1
B% = 2
SWAP A%, B%
PRINT A%; B%      ' 2 1
```

(第 3 章の練習 3 で、SWAP なしで入れ替える方法をやりました。
一時変数に避難させる、あの手です。SWAP はそれを 1 命令に
したものです。)

## 10.2 バブルソート — 隣どうしを見比べる

いちばん有名な入門ソートです。作戦:

> 隣どうしを見比べて、**左が大きかったら交換**。
> 端から端までやると、いちばん大きい値が右端に「浮かび上がる」。
> これを (右端を除いて) くり返す。

泡 (バブル) が水面に浮かぶ様子から名前がつきました。

```basic
' bubble.bas — バブルソート (途中経過つき)
DIM A%(7)
FOR I% = 0 TO 7
  READ A%(I%)
NEXT I%
DATA 64, 25, 12, 90, 8, 76, 33, 51

FOR PASS% = 0 TO 6                    ' 8 個なら 7 周で必ず終わる
  FOR I% = 0 TO 6 - PASS%             ' 確定した右端は見なくてよい
    IF A%(I%) > A%(I% + 1) THEN
      SWAP A%(I%), A%(I% + 1)
    END IF
  NEXT I%
  ' ---- 途中経過を表示 ----
  PRINT PASS% + 1; "周目:";
  FOR I% = 0 TO 7
    PRINT A%(I%);
  NEXT I%
  PRINT
NEXT PASS%
```

```text
 1 周目: 25  12  64  8  76  33  51  90 
 2 周目: 12  25  8  64  33  51  76  90 
 3 周目: 12  8  25  33  51  64  76  90 
 4 周目: 8  12  25  33  51  64  76  90 
 5 周目: 8  12  25  33  51  64  76  90 
 6 周目: 8  12  25  33  51  64  76  90 
 7 周目: 8  12  25  33  51  64  76  90 
```

途中経過を見てください。1 周目で 90 が右端に着地、2 周目で 76 が
その隣に…と、**右から順に確定**していくのが分かります。だから
内側のループは周回ごとに 1 つ短くできる (`6 - PASS%`) わけです。

4 周目で完成していますね。「1 周まるごと交換ゼロだったら、もう
完成」— そこで打ち切る改良が定番です (練習問題 2)。

## 10.3 選択ソート — チャンピオンを順に選ぶ

第 6 章の「最大値探し (暫定チャンピオン方式)」を覚えていますか?
あれをくり返すとソートになります。作戦:

> 残りの中から**最小**を探して、先頭と交換。
> 次は 2 番目の位置に、残りの最小を。以下くり返し。

```basic
' selection.bas — 選択ソート
DIM A%(7)
FOR I% = 0 TO 7
  READ A%(I%)
NEXT I%
DATA 64, 25, 12, 90, 8, 76, 33, 51

FOR I% = 0 TO 6
  ' A%(I%) 〜 A%(7) の中で最小の場所を探す (第 6 章の応用)
  MINPOS% = I%
  FOR J% = I% + 1 TO 7
    IF A%(J%) < A%(MINPOS%) THEN MINPOS% = J%
  NEXT J%
  SWAP A%(I%), A%(MINPOS%)   ' 最小を「今の先頭」へ
NEXT I%

FOR I% = 0 TO 7
  PRINT A%(I%);
NEXT I%
PRINT
```

```text
 8  12  25  33  51  64  76  90 
```

バブルとの違いは交換の回数です。バブルは見比べるたびに交換します
が、選択ソートは 1 周に 1 回だけ。「探すのはタダ、動かすのは高い」
という状況 (巨大なデータの並べ替えなど) では選択ソートが有利です。

## 10.4 挿入ソート — トランプの手札方式

トランプで手札を並べるとき、あなたはこうしているはず:

> 新しいカードを 1 枚取り、**すでに並んでいる手札の正しい位置に
> 差し込む**。

これが挿入ソートです。「差し込む」ために、大きいカードを
右へ 1 つずつずらして隙間を作ります:

```basic
' insertion.bas — 挿入ソート
DIM A%(7)
FOR I% = 0 TO 7
  READ A%(I%)
NEXT I%
DATA 64, 25, 12, 90, 8, 76, 33, 51

FOR I% = 1 TO 7                ' 0 番は「並んだ手札」の初期メンバー
  X% = A%(I%)                  ' 新しく取ったカード
  J% = I% - 1
  WHILE J% >= 0
    IF A%(J%) <= X% THEN EXIT WHILE   ' 差し込み位置が見つかった
    A%(J% + 1) = A%(J%)        ' 大きいカードを右へずらす
    J% = J% - 1
  WEND
  A%(J% + 1) = X%              ' 隙間に差し込む
NEXT I%

FOR I% = 0 TO 7
  PRINT A%(I%);
NEXT I%
PRINT
```

挿入ソートの美点は「**ほとんど並んでいるデータに速い**」こと。
すでに並んだデータなら、ずらす作業がゼロなので一瞬で終わります。
実務のライブラリでも「小さい範囲は挿入ソート」という形で
現役で使われています。

## 10.5 3 つのソートの比較

| ソート | 考え方 | 比較回数 | 特徴 |
|---|---|---|---|
| バブル | 隣と比べて浮かせる | O(n²) | いちばん素朴。教材の定番 |
| 選択 | 最小を選んで先頭へ | O(n²) | 交換が少ない |
| 挿入 | 正しい位置に差し込む | O(n²) | 並びかけのデータに強い |

3 つとも O(n²) — データが 10 倍になると手間は 100 倍です。
1 万件ならまだ一瞬ですが、100 万件になると目に見えて待たされます。

> **コラム: もっと速いソート**
> 実務で使われるクイックソートやマージソートは O(n log n) で、
> 100 万件でも一瞬です。核になるのは第 8 章の**再帰**:
> 「配列を半分に分けて、それぞれを (自分自身で) ソートして、
> 合わせる」という発想です。この本の範囲は超えますが、
> 「素朴なソートには限界がある」「その壁は再帰で破る」という
> 地図だけ持っておいてください。興味がわいたら examples/ に
> あるプログラムを改造して挑戦する価値があります。

## 10.6 文字列だってソートできる

比較 (`>`) が文字列にも使えるので、同じアルゴリズムがそのまま
名前の五十音順 (正確には文字コード順) 並べにも使えます:

```basic
' names.bas — 名前をバブルソート
DIM N$(4)
FOR I% = 0 TO 4
  READ N$(I%)
NEXT I%
DATA EMMA, CAROL, ALICE, DAVE, BOB

FOR PASS% = 0 TO 3
  FOR I% = 0 TO 3 - PASS%
    IF N$(I%) > N$(I% + 1) THEN SWAP N$(I%), N$(I% + 1)
  NEXT I%
NEXT PASS%

FOR I% = 0 TO 4
  PRINT N$(I%)
NEXT I%
```

```text
ALICE
BOB
CAROL
DAVE
EMMA
```

**アルゴリズムはデータの種類を選ばない** — 数値で覚えた手順が
文字列にそのまま効く。これがアルゴリズムを学ぶ意味です。

## 練習問題

1. バブルソートを「大きい順 (降順)」に変えてください。
   直すのは 1 文字です。
2. バブルソートに「1 周まるごと交換が無かったら打ち切る」改良を
   入れてください (ヒント: 交換したかを覚える変数を 1 周ごとに
   リセット)。
3. 20 個の乱数を作って配列に入れ、ソートしてから、第 9 章の
   二分探索で探す — という「フルコース」プログラムを書いて
   ください。
4. 成績表 (名前の配列と点数の配列のペア) を、**点数の高い順**に
   並べ替えてください。ただし名前と点数がバラバラにならないように
   (ヒント: 点数を SWAP するときは名前も SWAP)。
