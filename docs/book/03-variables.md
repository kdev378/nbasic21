# 第 3 章 覚えさせる — 変数

電卓と違って、プログラムは計算の途中結果を**覚えて**おけます。
覚えるための仕組みが**変数**です。プログラミングのすべての土台に
なる考え方なので、この章はゆっくり読んでください。

## 3.1 変数は「名前つきの箱」

変数とは、値をしまっておく**名前つきの箱**です。
箱に値を入れることを**代入**といい、`=` で書きます。

```basic
PRICE = 1980           ' PRICE という箱に 1980 を入れる
TAX = 1.1              ' TAX という箱に 1.1 を入れる
PRINT PRICE * TAX      ' 箱の中身を取り出して計算に使う
```

```text
 2178
```

大事なのは、`=` が数学の「等しい」では**ない**ことです。
`PRICE = 1980` は「PRICE と 1980 は等しい」という主張ではなく、
「**1980 を PRICE に入れろ**」という命令です。右のものを左に入れる。
矢印だと思ってください: `PRICE ← 1980`。

だから、数学では変な次の式も、プログラムでは意味が通ります:

```basic
COUNT = 10
COUNT = COUNT + 1      ' 今の COUNT (10) に 1 を足して、COUNT に入れ直す
PRINT COUNT            ' 11
```

「今の値に 1 足したものを、同じ箱に入れ直す」— つまり
**カウントアップ**です。これは全プログラミング言語で最頻出のイディオム
なので、目に焼き付けてください。

## 3.2 変数の名前のつけ方

- 使えるのは英字・数字・`_`。先頭は英字。大文字と小文字は
  区別されません (`price` と `PRICE` は同じ箱)。
- `PRINT` や `IF` などの命令語 (予約語) は使えません。
- **中身がわかる名前**をつけてください。`X` より `PRICE`、
  `A1` より `TOTAL` です。プログラムは書く時間より読む時間のほうが
  長いのです。

## 3.3 型 — 箱には種類がある

NBASIC-21 の箱 (変数) には 3 種類あります。

| 型 | 入るもの | 目印 (サフィックス) | 例 |
|---|---|---|---|
| 整数 (INTEGER) | …-2, -1, 0, 1, 2… | 名前の最後に `%` | `COUNT%` |
| 実数 (DOUBLE) | 小数を含む数 | なし (または `#`) | `PRICE` |
| 文字列 (STRING) | 文字のならび | 名前の最後に `$` | `NAME$` |

名前の最後の記号で型が決まります。**記号まで含めて名前**なので、
`A%` と `A$` と `A` は別々の箱です。

```basic
COUNT% = 3                 ' 整数の箱
PI = 3.14159               ' 実数の箱 (目印なし = 実数)
NAME$ = "山田"             ' 文字列の箱
PRINT NAME$; "さんは"; COUNT%; "回目、円周率は"; PI
```

```text
山田さんは 3 回目、円周率は 3.14159
```

使い分けの目安はシンプルです:

- **数を数える**もの (回数、個数、番号) → `%` の整数
- **量をはかる**もの (金額、距離、平均) → 目印なしの実数
- **文字** → `$`

文字列の箱に数を入れようとするとエラーになります。
箱の種類が違うからです:

```text
error: 代入: INTEGER を STRING に変換できません
```

## 3.4 文字列をつなぐ

文字列は `+` または `&` でつなげられます。`&` は数値を自動で
文字に変換してくれるので便利です:

```basic
FAMILY$ = "山田"
GIVEN$ = "太郎"
FULL$ = FAMILY$ + " " + GIVEN$
PRINT FULL$
AGE% = 20
PRINT FULL$ & " (" & AGE% & "歳)"
```

```text
山田 太郎
山田 太郎 (20歳)
```

## 3.5 INPUT — キーボードから受け取る

ここまでのプログラムは毎回同じ結果でした。`INPUT` を使うと、
**実行するたびに違う値**を人間から受け取れます。

```basic
INPUT "お名前は"; NAME$
INPUT "年齢は"; AGE%
PRINT "こんにちは、"; NAME$; "さん!"
PRINT "来年は"; AGE% + 1; "歳ですね。"
```

実行すると、プログラムは `?` を出して**入力を待ちます**。
文字を打って Enter を押すと続きが動きます:

```text
お名前は? 山田
年齢は? 20
こんにちは、山田さん!
来年は 21 歳ですね。
```

`INPUT "表示する文句"; 変数` の形で覚えてください。
これで「対話するプログラム」が書けるようになりました。

## 3.6 実践: 割り勘計算機

この章の知識だけで、もう実用品が作れます:

```basic
' warikan.bas — 割り勘計算機
INPUT "合計金額は"; TOTAL
INPUT "何人で割る"; N%
PERPERSON% = INT(TOTAL / N%) ' INT() は小数点以下の切り捨て
AMARI% = TOTAL - PERPERSON% * N%
PRINT "ひとり"; PERPERSON%; "円"
PRINT "幹事が余分に払う分:"; AMARI%; "円"
```

```text
合計金額は? 10000
何人で割る? 3
ひとり 3333 円
幹事が余分に払う分: 1 円
```

新顔の `INT()` は「小数点以下を切り捨てる」**組込関数**です。
関数についてはこの先の章で詳しくやりますが、
「`INT(なにか)` と書くと切り捨てた結果になる」とだけ覚えれば
今は十分です。

## 3.7 変数の中身は「実行のたびにリセット」

プログラムが終わると、変数の中身はすべて消えます。
次に実行したときは、また 0 (文字列なら空っぽ) からのスタートです。
「覚えたことをファイルに残す」方法は第 12 章で学びます。

## 他の言語では

「名前つきの箱に値を入れる」考え方は万国共通です。

```text
NBASIC-21 :  AGE% = 20
Python    :  age = 20
JavaScript:  let age = 20;
C         :  int age = 20;
```

型を名前の記号で表すのは BASIC 独特ですが、「整数と実数と文字列は
別物」という区別自体は C や Java など多くの言語がもっと厳格に
持っています。ここで慣れておくと後で楽です。

## 練習問題

1. 縦と横の長さを `INPUT` で聞いて、長方形の面積と周の長さを
   表示するプログラムを書いてください。
2. 摂氏温度 (°C) を聞いて、華氏温度 (°F) に変換するプログラムを
   書いてください。変換式は「華氏 = 摂氏 × 1.8 + 32」です。
3. 次のプログラムを実行すると何が表示されるか、**実行せずに**
   紙で予想してから、実行して確かめてください:

   ```basic
   A% = 5
   B% = 3
   A% = A% + B%
   B% = A% - B%
   A% = A% - B%
   PRINT A%; B%
   ```

   (ヒント: 2 つの箱の中身がどうなったか、1 行ずつ表に書くと
   見えてきます。これは有名なテクニックです。)
