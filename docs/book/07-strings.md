# 第 7 章 文字をあやつる

コンピュータの仕事の半分は、実は数値計算ではなく**文字の処理**です。
検索、置換、切り出し、整形 — この章では文字列を自由に切り貼りする
道具を手に入れます。

> **ことわり**: この章のテクニックは半角英数字で練習します。
> 日本語 (全角文字) は内部で 1 文字が複数バイトになるため、
> 「N 文字目」の切り出しがずれることがあります。文字列の中身を
> いじるときは英数字で、表示するだけなら日本語 OK、と覚えて
> ください。

## 7.1 文字列の道具箱

| 関数 | 意味 | 例 | 結果 |
|---|---|---|---|
| `LEN(s$)` | 長さ (文字数) | `LEN("HELLO")` | 5 |
| `LEFT$(s$, n)` | 左から n 文字 | `LEFT$("HELLO", 2)` | "HE" |
| `RIGHT$(s$, n)` | 右から n 文字 | `RIGHT$("HELLO", 3)` | "LLO" |
| `MID$(s$, i, n)` | i 文字目から n 文字 | `MID$("HELLO", 2, 3)` | "ELL" |
| `INSTR(s$, t$)` | t$ が何文字目にあるか (無ければ 0) | `INSTR("HELLO", "LL")` | 3 |
| `UCASE$(s$)` / `LCASE$(s$)` | 大文字化 / 小文字化 | `UCASE$("abc")` | "ABC" |
| `STR$(n)` / `VAL(s$)` | 数→文字列 / 文字列→数 | `VAL("42")` | 42 |

位置は **1 文字目から**数えます (配列の 0 始まりと違うので注意)。
まず遊んでみましょう:

```basic
S$ = "PROGRAMMING"
PRINT LEN(S$)              ' 11
PRINT LEFT$(S$, 3)         ' PRO
PRINT MID$(S$, 4, 4)       ' GRAM
PRINT INSTR(S$, "GRAM")    ' 4
PRINT LCASE$(S$)           ' programming
```

## 7.2 1 文字ずつ調べる — 文字列版ループ

`MID$(s$, i, 1)` で「i 文字目の 1 文字」が取れます。
これを FOR と組み合わせると、文字列を 1 文字ずつ調べられます。
配列を 1 個ずつ調べたのと同じ発想です:

```basic
S$ = "BANANA"
FOR I% = 1 TO LEN(S$)
  PRINT I%; "文字目:"; MID$(S$, I%, 1)
NEXT I%
```

応用: 文字を**数え**ます。「BANANA に A は何個?」

```basic
S$ = "BANANA"
COUNT% = 0
FOR I% = 1 TO LEN(S$)
  IF MID$(S$, I%, 1) = "A" THEN COUNT% = COUNT% + 1
NEXT I%
PRINT "A は"; COUNT%; "個"
```

```text
A は 3 個
```

第 6 章のカウントと完全に同じ型です。「並んだものを順に見て、
条件に合うものを数える」— 対象が配列でも文字列でも、
アルゴリズムは変わりません。

## 7.3 逆から作る — 文字列の反転

文字列をひっくり返します。累積パターンの文字列版で、
「空の文字列に、1 文字ずつ**前に**つなげていく」のがコツ:

```basic
S$ = "STRESSED"
R$ = ""                        ' 空っぽからスタート
FOR I% = 1 TO LEN(S$)
  R$ = MID$(S$, I%, 1) + R$    ' 新しい文字を先頭にくっつける
NEXT I%
PRINT S$; " を逆にすると "; R$
```

```text
STRESSED を逆にすると DESSERTS
```

(ストレスも逆から読めばデザートです。)

反転ができると**回文判定** (「しんぶんし」のように逆から読んでも
同じか) は簡単です: 反転した結果と元を `=` で比べるだけ。

## 7.4 文字コード — 文字の正体は番号

コンピュータの中では、文字は**番号**で管理されています
(半角英数字の番号表を ASCII コードといいます)。
`ASC` で文字→番号、`CHR$` で番号→文字に変換できます:

```basic
PRINT ASC("A"); ASC("B"); ASC("Z")    '  65  66  90
PRINT ASC("a"); ASC("0")              '  97  48
PRINT CHR$(65); CHR$(66); CHR$(67)    ' ABC
```

「A=65 から Z=90 まで連番」— この規則性が使えます。
たとえばアルファベット表を一発生成:

```basic
FOR C% = ASC("A") TO ASC("Z")
  PRINT CHR$(C%);
NEXT C%
PRINT
```

```text
ABCDEFGHIJKLMNOPQRSTUVWXYZ
```

## 7.5 実践: シーザー暗号

文字コードの応用で、2000 年前にカエサル (シーザー) が使ったと
される暗号を作ります。仕組みは「各文字をアルファベット順で
3 つ後ろにずらす」だけ。A→D、B→E、…、X→A (最後は先頭に戻る)。

「ずらして、はみ出したら先頭に戻る」— `MOD` の出番です:

```basic
' caesar.bas — シーザー暗号 (大文字英字のみ)
INPUT "メッセージ (大文字英字)"; S$
INPUT "ずらす数 (1〜25)"; K%

R$ = ""
FOR I% = 1 TO LEN(S$)
  C% = ASC(MID$(S$, I%, 1))
  IF C% >= ASC("A") AND C% <= ASC("Z") THEN
    ' A を 0 番として K% ずらし、26 で折り返してから A に戻す
    C% = (C% - ASC("A") + K%) MOD 26 + ASC("A")
  END IF
  R$ = R$ + CHR$(C%)
NEXT I%
PRINT "暗号文: "; R$
```

```text
メッセージ (大文字英字)? HELLO WORLD
ずらす数 (1〜25)? 3
暗号文: KHOOR ZRUOG
```

英字以外 (スペースなど) はそのまま通しています。
復号 (元に戻す) は、ずらす数を `26 - K%` にして同じプログラムに
かけるだけ。試してみてください。

## 7.6 実践: 単語に分ける

「スペース区切りの文を単語に分ける」は、文字列処理の登竜門です。
考え方: 文字を順に見て、スペースが来るたびに「そこまでの
たまり」をひとつの単語として確定する。

```basic
S$ = "TO BE OR NOT TO BE"
DIM WORDS$(19)
N% = 0                          ' 見つけた単語の数
W$ = ""                         ' 今たまっている途中の単語
FOR I% = 1 TO LEN(S$)
  C$ = MID$(S$, I%, 1)
  IF C$ = " " THEN
    IF W$ <> "" THEN            ' たまっていれば単語として確定
      WORDS$(N%) = W$
      N% = N% + 1
      W$ = ""
    END IF
  ELSE
    W$ = W$ + C$                ' スペース以外はためる
  END IF
NEXT I%
IF W$ <> "" THEN                ' 最後の単語を忘れずに
  WORDS$(N%) = W$
  N% = N% + 1
END IF

PRINT N%; "単語:"
FOR I% = 0 TO N% - 1
  PRINT I%; ": "; WORDS$(I%)
NEXT I%
```

「最後の単語を忘れずに」の 3 行に注目。文末にはスペースが
無いので、ループが終わった時点でまだ W$ に単語が残っています。
こういう**端っこの処理** (境界条件) を丁寧に拾えるかどうかが、
動くプログラムと動かないプログラムの分かれ目です。

## 他の言語では

```text
NBASIC-21 :  LEN(S$)   MID$(S$, 2, 3)   INSTR(S$, "LL")
Python    :  len(s)    s[1:4]           s.find("LL") + 1
JavaScript:  s.length  s.substr(1, 3)   s.indexOf("LL") + 1
```

Python などは位置を 0 から数えるので +1/-1 のずれがありますが、
「長さ・切り出し・検索」の 3 点セットはどこでも同じです。
7.6 の単語分割は、多くの言語では `split` という関数一発ですが、
その `split` の中身がまさに今書いたループなのです。

## 練習問題

1. 名前をフルネーム (例: `TARO YAMADA`) で入力させ、
   スペースの位置を `INSTR` で探して、名と姓を別々の行に
   表示してください。
2. 入力された文字列が回文かどうか判定してください (7.3 参照)。
3. 入力された文字列に含まれる母音 (A, E, I, O, U) の数を
   数えてください。大文字でも小文字でも数えられるようにすると
   なお良し (ヒント: 先に `UCASE$` してしまう)。
4. シーザー暗号を改造して、ずらす数 1〜25 の全パターンを一気に
   表示する「総当たり解読機」を作ってください。暗号文
   `WKLV LV D VHFUHW` を解読してみましょう。
